## CRUD scaffolding playbook

Use this checklist whenever you add a new entity so it stays consistent with the existing flow (Organizations, Pets, etc.).

1) Domain types  
- Define the `Entity` type from the Drizzle table (`$inferSelect`).  
- Define `CreateEntityData = Omit<$inferInsert, 'id'>` (strip autogenerated columns).  
- Define `UpdateEntityData = Partial<CreateEntityData>`.  
- Add a `Filters` type for list queries.

2) Repository interface  
- Extend `CrudRepository<Entity, CreateEntityData, UpdateEntityData, Filters>`.  
- Add any domain-specific reads (e.g., `findBySlug`, `findByEmail`).

3) Repository implementation (Postgres preferred)  
- Start from `createDrizzleRepository` for base CRUD.  
- Add a filter builder (`buildWhereConditions`) and override `find` / `count` to respect filters.  
- Override `create` to coerce optionals (e.g., `undefined` → `null`) or inject defaults.  
- Name exports consistently (`postgresXRepository`) and surface via the factory (`getXRepository`).
  - Pets are currently memory-backed but still use `getPetRepository`; swap the factory to a Postgres repo once a `pets` table exists.

4) Server actions  
- Thin adapters that: check auth, enforce uniqueness/domain rules, call the repo, and `revalidatePath` for affected routes.  
- Actions accept the narrow `Create/Update` input (no `id`/timestamps) and add `createdBy/updatedBy` using the current user.

5) UI (forms, lists, shells)  
- Forms use `react-hook-form` + resolver schemas; submit to server actions; show toasts; navigate on success.  
- Lists/tables consume `PaginatedResult` from `find` and reuse the shared pagination component.  
- Shell components receive the domain user and pass it to headers/sidebars; utilities (`getUserDisplayName`, `getUserInitials`) are null-safe.

Migration note for Pets  
- The pet flow currently uses the in-memory repository (`getPetRepository`) that already honors `CrudRepository` and filtering. When a `pets` table lands, add a Postgres implementation with `createDrizzleRepository`, wire filters, then point `getPetRepository` in the factory to the new repo.
